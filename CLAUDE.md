# CLAUDE.md — 니트커피(Knit Coffee) 도매 주문 시스템

> 이 문서는 Claude Code가 프로젝트를 이해하기 위해 매 세션 자동으로 읽는 컨텍스트 파일입니다.
> 기존 개발 과정에서 확정된 규칙과 정책을 담고 있으니, 코드 수정 시 아래 규칙을 반드시 준수하세요.

---

## 1. 프로젝트 개요

- **이름:** 니트커피(Knit Coffee) 도매 주문 시스템 (웹 앱)
- **운영자:** 이강민 (니트커피 대표, 개인사업자)
- **목적:** 도매 거래처의 원두 주문 접수 + 샘플 신청 + 경영/재무 관리 + OEM 공장(클라리멘토) 채무 관리를 하나의 웹 앱으로 통합
- **로스터리:** 원두는 '클라리멘토'를 통해 OEM 생산·납품

---

## 2. 기술 스택

- **백엔드:** Express.js (Node.js)
- **프론트엔드:** React
- **디자인:** Tailwind CSS
- **데이터베이스:** SQLite (파일명: `data.db`, 프로젝트 루트에 위치)
- **배포:** GitHub → Railway 자동 배포 (push 시 자동 반영)
- **알림:** 주문 발생 시 백엔드에서 실시간 주문 알림 이메일 직접 발송

---

## 3. 사용자 권한 구조 (중요 — 용어 주의)

과거 대화에서 용어 혼동이 있었으므로 아래 정의를 정확히 따르세요.

- **소유자(Owner):** 이강민 대표. 모든 기능 접근 가능. 경영 대시보드 열람 가능.
- **관리자(Admin/직원, = 기존 대화의 '매니저'):** 기록·입력만 가능. 경영 대시보드는 열람 불가.
- 경영 대시보드는 **소유자만** 볼 수 있고, 관리자는 데이터 **기록만** 가능.

---

## 4. 도매 주문 규칙

- **최소 주문:** 5kg 이상
- **5kg 판정 기준:** "원두 카테고리" 상품의 **주문 수량 합계**로 판단 (중량 환산 없이 개수 기준, **샘플 제외**)
- **관리자 대리 주문:** 소유자 + 관리자(직원)가 입력 가능. 입력 즉시 상태를 '처리완료'로 하여 클라리멘토 자동발주까지 연결.

---

## 5. 샘플 신청 시스템

- **가격:** 무료 (승인된 거래처에 한해 **1회만**)
- **신청 제약:** 원두 **최대 2종**, 각 **1kg씩**만 신청 가능
- **신청 페이지:** 별도의 샘플 전용 신청 페이지 존재 (일반 주문과 분리)
- **전환 인센티브:** 없음 (샘플 주문만 깔끔하게 처리)
- **주문 식별:** 관리자 주문 목록에서 "샘플" 뱃지로 표시 + 필터링 가능

### 샘플 신청 승인 흐름 (2갈래)
1. **사업자등록 O (정상 업체):** 가입 → 자동승인 → 바로 무료 샘플 1회 신청 가능
2. **사업자등록 X (미등록 업체):** 가입은 되지만 자동승인 안 됨 → 관리자에게 승인 요청 → 소유자가 직접 승인 → 이후 무료 샘플 1회 신청 가능

- **판별 기준:** 가입 시 사업자등록번호 입력란. 채워져 있으면 자동승인 / 비어 있으면 승인 대기.
- **검증:** 사업자등록번호 형식 검증 필요.

---

## 6. 재무 / 경영 관리

- **매출 입력:** 매장 매출은 일별 수기 입력
- **재무 카테고리 분류:** 지출·매출 입력 시 다음으로 구분
  - 니트커피 매장
  - 홀세일(도매)
  - 온라인 판매
  - 기타 지출
- **카드 수수료:** (미확정 — 이강민님과 재논의 필요. 과거 "아예 안 다루는 걸로" 방향으로 정리된 바 있음)

### OEM 공장(클라리멘토) 채무 관리 — 관리자 전용 화면 4개
- 기존 '거래처 미수금' 화면 구조를 공장 쪽으로 미러링한 형태
- 4개 화면 모두 관리자(소유자) 전용, 직접 입력·수정 방식

---

## 7. 외부 연동

- **카카오 로그인:** 카카오 개발자 앱 연동 진행 중 (Redirect URI 설정 등)
- **카카오 알림톡:** 주문 알림. 우선은 관리자(이강민)에게 발송하는 것부터 시작하고,
  거래처 고객 대상 알림톡은 심사·비용 확인 후 추후 확장.
